# 异环 · 喵呜快报 活动流水查询

通过完美世界客服系统接口查询「喵呜快报」活动道具流水，并对抽卡消耗/获得做统计。

## 背景

「喵呜快报」活动：消耗方斯购买读物（附赠刮刮卡），刮刮卡开奖获得方斯。本目录脚本用于抓取活动流水并计算收益。

## 文件

| 文件 | 说明 |
| --- | --- |
| `raw-request.js` | 浏览器导出的原始 fetch 请求样本（含 cookie 与 body 签名结构） |
| `fetch-all.js` | 分批抓取脚本，7 天一段，合并所有记录到 `merged.json` |
| `stat.js` | 统计脚本，输出按读物统计、按奖金档、汇总（消耗/获得/流水/收益） |
| `merged.json` | `fetch-all.js` 产出的合并结果（1084 条） |

## 数据结构

`merged.json` 为数组，每条记录三个字段：

- `logTime` — 流水时间（分钟级，如 `2026-07-03 12:21`）
- `scratchCardId` — 读物名（决定消耗多少方斯）
- `award` — 获得奖励，格式 `方斯*<数字>`，空代表 0

## 使用方法

1. 从浏览器（登录 `kf.wanmei.com` 后）导出原始请求，更新 `raw-request.js` 与 `fetch-all.js` 顶部的 `cookie`（尤其 `SESSION` 会过期）。
2. 换账号时同步改 `fetch-all.js` 里的 `roleId`（当前 `219012188065`）。
3. 改时间范围改 `fetch-all.js` 顶部 `START` 常量。
4. 运行：

```bash
cd yihuan-miaoou-flow
node fetch-all.js   # 抓取，生成 merged.json
node stat.js        # 统计，读 merged.json
```

## 请求结构要点

- 接口：`POST https://kf.wanmei.com/selfItemFlowQuery/search`
- `Content-Type: multipart/form-data`，body 分两段：
  - 具名业务字段：`typeId`/`gameId`/`roleId`/`itemType`/`itemSubType`/`item5`/`startTime`/`endTime`/`pageSize`/`pageNo` 等
  - `0`~`N` 数字命名字段，值按序拼接 = 上述字段的 URL-encoded query string。**疑似防篡改签名，改任何字段必须同步重算这整块，否则请求被拒。**
- 服务端限制：`startTime` 与 `endTime` 间隔 ≤ 7 天，超过返回 `code=1`「每次查询日期间隔不能大于7天！请重新选择！」
- `pageSize=1000` 在单段 7 天内足够查完。

## 统计口径

- **消耗**：`scratchCardId` 按读物售价计
- **获得**：`award` 解析 `方斯*N` 的 `N`，空 = 0
- **流水** = 总消耗 + 总获得
- **收益** = 总获得 − 总消耗
